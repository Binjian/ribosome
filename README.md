# cell-ribosome

`cell-ribosome` is an nbdev-based Python project for turning Markdown documents into a richer intermediate representation:

- Pandoc JSON AST
- reorganized section/slide structure
- semantic summaries for document nodes
- vector embeddings stored in ChromaDB

The current implementation is centered around the `DOM` model in `cell_ribosome/dom.py`.

## What It Does

Given a Markdown file, the project can:

1. Load and convert the file into a Pandoc JSON AST.
2. Reorganize headers into nested `Section` nodes.
3. Optionally split slide-style Markdown into section-like slide nodes.
4. Summarize text blocks and image nodes with LLM clients.
5. Generate embeddings for summarized nodes and persist them to ChromaDB.

## Requirements

The code expects the following runtime pieces to be available:

- Python 3.7+
- Pandoc installed on the machine
- an Ollama-compatible endpoint for image summarization and embeddings
- an OpenAI-compatible endpoint for text summarization
- local environment variables loaded from `.env`

Environment variables used by the code:

```bash
OLLAMA_API_KEY=...
DASHSCOPE_API_KEY=...
```

## Core API

### `DOM`

The main entry point is the `DOM` class in [cell_ribosome/dom.py](/Users/x/devel/cell/ribosome/cell_ribosome/dom.py).

Typical flow:

1. Construct `DOM` with a Markdown file path.
2. Run `setup()` to load Markdown and generate or reload AST artifacts.
3. Run `await textualize()` to build semantic summaries.
4. Run `await embed()` to create embeddings and store them in ChromaDB.

### Main functions

- `DOM.setup()`: loads markdown, generates Pandoc JSON, initializes artifact paths.
- `DOM.reorg()`: converts header blocks into nested `Section` structures.
- `DOM.reorg_slides()`: handles slide-delimited markdown.
- `DOM.textualize()`: walks the AST and adds semantic summaries.
- `DOM.embed()`: creates embeddings and writes `_embed.json`.
- `analyze_one_document_async()`: async helper for one-file semantic analysis.
- `embed_one_document_async()`: async helper for one-file embedding.

## Example

```python
import asyncio
from pathlib import Path

from cell_ribosome.dom import DOM


async def main():
    doc = DOM(Path("notes.md"))
    doc.setup()

    await doc.textualize()
    await doc.embed()

    print(doc.ast_json_file)
    print(doc.semantics_json_file)
    print(doc.embed_json_file)


asyncio.run(main())
```

## Generated Artifacts

For a source file like `notes.md`, the code writes neighboring artifacts such as:

- `notes_ast.json`
- `notes_semantics.json`
- `notes_embed.json`

These are used as cacheable intermediate outputs for later processing.

## Current Caveats

At the moment, the repository looks more like an active prototype than a polished package. In particular:

- packaging metadata mixes `ribosome` and `cell_ribosome`
- import-time side effects print environment variables and initialize clients
- dependency lists include duplicates and development tools in runtime requirements
- there is no committed test suite yet
