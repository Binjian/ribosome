# CLAUDE.md

## Project Overview

**cell-ribosome** is a Python library for intelligent document processing. It transforms Markdown documents into enriched intermediate representations using LLM-powered semantic analysis and vector embeddings.

Pipeline: Markdown → Pandoc JSON AST → reorganized sections → LLM summaries → ChromaDB vector embeddings

## Tech Stack

- **Language:** Python 3.7+
- **Build/Dev framework:** nbdev (notebook-driven development)
- **Package manager:** setuptools (via `setup.py` + `settings.ini`)
- **Doc generation:** Quarto
- **Key libraries:** Pydantic, pypandoc, ollama, openai (DashScope-compatible), langchain, chromadb, pymupdf

## Project Structure

```
ribosome/              # Main Python package (lib_path from settings.ini)
nbs/                   # Source notebooks — primary source of truth
  00_core.ipynb        # Core module
  01.core.dom.ipynb    # DOM module (main logic)
scripts/               # Standalone scripts (tree-sitter parser, tests)
res/                   # Resources: PDFs, sample data
db/                    # ChromaDB persistent storage
_proc/                 # Generated Quarto output
```

## Development Workflow

This is an **nbdev project**. Source code lives in Jupyter notebooks under `nbs/`. Python modules in `ribosome/` are **autogenerated** — do not edit them directly.

```sh
# Install in development mode
pip install -e .

# After editing notebooks, export to Python modules + run tests + clean notebooks
nbdev_prepare

# Individual steps:
nbdev_export      # Export notebooks to ribosome/
nbdev_test        # Run notebook tests
nbdev_clean       # Clean notebook metadata
```

## Build & Test

```sh
# Install dependencies
pip install -e .

# Run nbdev test suite
nbdev_test

# Run shell-based tests
bash scripts/test_all.sh
```

## External Requirements

- **Pandoc** must be installed and on PATH (used by pypandoc)
- **Ollama** service must be running for image summarization and embeddings
- **Environment variables** (via `.env`):
  - `OLLAMA_API_KEY` — Ollama endpoints
  - `DASHSCOPE_API_KEY` — Alibaba DashScope API (text summarization)
  - `GEMINI_API_KEY` — Google Gemini (optional)
  - `MINIMAX_API_KEY` — Minimax API (optional)

## Key Conventions

- **Do not edit `ribosome/*.py` directly** — they are autogenerated by nbdev from `nbs/`
- All LLM calls are **async** (`AsyncClient`, `AsyncOpenAI`)
- Pydantic `BaseModel` for data classes
- Bilingual prompts (Chinese/English) in LLM calls
- Artifact caching: `{filename}_ast.json`, `{filename}_semantics.json`, `{filename}_embed.json`
- `#| export` cell directive in notebooks marks code for export to Python modules
- `#| notest` flag skips cells during `nbdev_test`

## CI/CD

- GitHub Actions workflows in `.github/workflows/`
  - `test.yaml` — runs nbdev3 CI tests
  - `deploy.yaml` — deploys Quarto docs to GitHub Pages
- Deploys on push to `main`, `master`, or `dom_module` branches
